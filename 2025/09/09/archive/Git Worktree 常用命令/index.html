Git Worktree 常用命令

发表于2025-09-09|更新于2025-09-09
|浏览量:

创建 Worktree

基于现有分支创建

应用场景：当需要同时工作在两个不同分支上，比如一个分支用于开发新功能，另一个分支用于修复紧急 bug，无需来回切换分支。

创建新分支并检出

应用场景：需要从特定分支或提交创建新分支进行功能开发或问题排查，同时保持主工作区不受影响。

列出所有 Worktree

应用场景：查看当前所有活跃的工作树，了解各个工作树的状态、所在分支和位置，便于管理和切换。

删除 Worktree

应用场景：完成特定功能开发或问题修复后，清理不再需要的工作树，释放磁盘空间。

锁定工作树（防止被意外删除）

应用场景：1.保护长期存在的功能分支工作树; 2.保护用于 CI/CD 或自动化构建的工作树;3. 在共享存储或网络位置上使用工作树; 4. 防止自己误操作

清理工作树

应用场景：你用正确的方式 (git worktree remove) 删除工作树，你永远不需要 prune。但如果你不小心直接删了文件夹，就需要 prune 来帮 Git 收拾烂摊子。
