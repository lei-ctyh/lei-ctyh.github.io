Git 入门指南：写给完全不懂版本控制的你

发表于2025-12-02|更新于2026-03-11|技术教程
|浏览量:

前言

如果你刚接触编程，可能会有这样的经历：

  • 改代码改崩了，想恢复昨天的版本，却发现没备份
  • 和同事一起开发，不知道怎么合并各自的修改
  • 听别人说 “commit”、”push”、”rebase”，完全听不懂

这篇文章用最通俗的语言，帮你搞懂 Git 是什么、怎么用。

一、Git 是什么？

一句话解释：Git 是一个”代码时光机”，能帮你：

  • 保存代码的每一个历史版本
  • 随时穿越回任意时间点
  • 多人协作开发不打架

二、核心概念

2.1 仓库（Repository）

仓库就是一个被 Git 管理的项目文件夹。

重点：

  • 工作目录 = 你能看到和编辑的文件
  • 本地仓库 (.git) = 隐藏的”存档库”，存着所有历史版本

💡 只要 .git 文件夹不删，你就能恢复到任意一次 commit 的状态。

2.2 代码的三个状态

flowchart LR
    A[工作目录<br/>Working Dir<br/>你正在改的文件] -->|git add| B[暂存区<br/>Staging Area<br/>准备要提交的]
    B -->|git commit| C[本地仓库<br/>Local Repo<br/>已保存的历史]
状态 说明 类比
工作目录 你正在编辑的文件 正在写的作业
暂存区 准备提交的修改 放进书包准备交的作业
本地仓库 已保存的历史记录 老师收走存档的作业

2.3 本地 vs 远端

概念 说明
本地（Local） 你自己电脑上的仓库
远端（Remote） 服务器上的仓库（如 GitHub、GitLab）

它们之间的同步：

flowchart LR
    A[本地仓库] -->|git push<br/>推送| B[远端仓库]
    B -->|git pull<br/>拉取| A

三、分支（Branch）

3.1 什么是分支？

分支就是代码的”平行世界”。

想象一下：你在写一个网站，老板突然说要加个新功能。你不想在正式代码上直接改（怕改崩），于是：

  1. 创建一个新分支 feature-xxx
  2. 在新分支上随便折腾
  3. 测试没问题后，合并回主分支
gitGraph
    commit id: "A"
    commit id: "B"
    commit id: "C"
    branch feature
    commit id: "D"
    commit id: "E"
    checkout main
    merge feature id: "F"

3.2 常见分支

分支名 用途
master / main 正式版本，稳定的代码
develop 开发中的代码
feature-xxx 某个新功能的开发
hotfix-xxx 紧急修复 bug

四、Merge vs Rebase

这是两种合并代码的方式，效果相同但历史记录不同。

Merge（合并）

gitGraph
    commit id: "D"
    commit id: "E"
    commit id: "F"
    commit id: "G"
    branch feature
    checkout feature
    commit id: "A"
    commit id: "B"
    commit id: "C"
    checkout main
    merge feature id: "H (合并节点)"
  • 会产生一个”合并节点”
  • 历史记录能看出分支合并过程

Rebase（变基）

gitGraph
    commit id: "D"
    commit id: "E"
    commit id: "F"
    commit id: "G"
    commit id: "A'"
    commit id: "B'"
    commit id: "C'"
  • 把你的提交”嫁接”到最新节点后面
  • 历史记录更整洁（一条直线）

六、常见问题

Q1：修改了但没 commit，能恢复吗？

情况 能否恢复
已 commit ✅ 永久保存，随时可恢复
只 add 没 commit ⚠️ 有办法但麻烦
没 add 也没 commit ❌ 丢了就是丢了

Q2：.git 文件夹能删吗？

不能！ 删了就等于删掉所有历史记录，只剩当前版本的文件。

Q3：push 和 pull 的区别？

  • push：你 → 服务器（上传）
  • pull：服务器 → 你（下载）

七、总结

Git 的核心就三件事：

  1. 保存历史：每次 commit 就是一次存档
  2. 分支开发：在平行世界里随便折腾
  3. 多人协作：通过 push/pull 同步代码

记住这张图：

flowchart LR
    A[工作目录] -->|add| B[暂存区]
    B -->|commit| C[本地仓库]
    C -->|push| D[远端仓库]
    D -->|pull| C

入门建议：先把 add、commit、pull、push 用熟，其他命令遇到再学。
