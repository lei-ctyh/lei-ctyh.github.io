开发前准备

在开始之前，请确保您的本地环境已安装以下工具：

  • **Git**：版本控制工具。
  • GitHub 账号：进行项目托管与版本控制。
  • **Node.js**：搭建本地开发环境。

创建您的沙箱仓库

强烈建议不要在正在记笔记的仓库中进行任何开发操作，以避免数据丢失的风险。

打开终端，切换到刚刚创建的仓库路径

如果不存在 plugins 文件夹，请创建它

在执行项目生成时，系统会提示您填写一些与项目相关的信息。请参考下表填写相应的值：

属性 类型 必需 描述
author string 是 插件作者的名字
description string 是 插件的详细描述
id string 是 插件的唯一标识
name string 是 插件的显示名称
isDesktopOnly boolean 否 插件是否使用 NodeJS 或 Electron API。默认值为 false。
minAppVersion string 否 插件所需的最低 Obsidian 版本，默认值为 ‘0.15.0’。
version string 否 插件的版本，默认值为 ‘1.0.0’。
authorUrl string 否 指向您个人网站的 URL。
fundingUrl string 否 资助信息的 URL。

注意：在项目生成时，系统会远程拉取 GitHub 上的项目，请确保您已在 GitHub 配置 SSH KEY。

在插件开发的过程中，您可能熟悉经常需要重新启动/重新加载 Obsidian，或者进入其插件设置以禁用和重新启用您正在使用的插件。这是一个麻烦的过程，你可以使用 hot-reload 插件。
