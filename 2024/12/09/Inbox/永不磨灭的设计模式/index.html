适配器模式

适配器模式（Adapter Pattern），又称包装样式、包装器（Wrapper）或变压器模式，是一种结构型设计模式。它允许接口不兼容的类一起工作，通过将一个类的接口转换成客户端所期望的另一种接口来实现。以下是关于适配器模式的详细解释：

一、基本概念

  • 目标角色（ITarget）：期望的接口，即客户端希望调用的接口。
  • 适配器（Adapter）：将 Adapted（被适配者）转换为目标角色 ITarget 的类实例。适配器类负责实现目标接口，并在其方法内部调用被适配者的相关方法。
  • Adapted（被适配者）：需要适配的类，其接口与目标接口不兼容。

二、主要类型

适配器模式主要分为三种类型：类适配器模式、对象适配器模式和接口适配器模式。

  1. 类适配器模式

    • 通过继承被适配者类（src 类）并实现目标接口（dst 接口）来完成适配。
    • 由于 Java 是单继承机制，类适配器要求目标接口必须是接口，这限制了其使用范围。
    • 适配器类可以重写被适配者类的方法，以增强适配的灵活性。

  2. 对象适配器模式

    • 通过持有被适配者类的实例（对象）并实现目标接口来完成适配。
    • 解决了类适配器必须继承被适配者类的局限性问题，不再要求目标接口必须是接口。
    • 使用成本更低，更灵活，符合合成复用原则。

  3. 接口适配器模式（缺省适配器模式）

    • 适用于一个接口定义了多个方法，但只需要实现其中一部分方法的情况。
    • 通过提供一个抽象类实现接口的所有方法（但实现为空方法），然后继承该抽象类并重写需要的方法。
    • 使得子类可以有选择地实现接口中的方法，而不是必须实现所有方法。

三、工作原理

适配器模式的工作原理是将一个类的接口转换成另一种接口，让原本接口不兼容的类可以兼容。从用户的角度来看，被适配者是被隐藏的，用户调用的是适配器转化出来的目标接口方法。适配器再调用被适配者的相关接口方法，并将结果返回给用户。用户感觉只是和目标接口交互，而看不到被适配者。

四、适用场景

  1. 接口不兼容：当系统中的某个类的接口与客户端期望的接口不兼容时，可以使用适配器模式将其转换为客户端期望的接口。
  2. 类的复用：当需要复用一些现有的类，但其接口与系统要求的接口不匹配时，可以通过适配器模式将这些类适配到系统中。
  3. 封装有关适配的逻辑：当需要封装与具体业务逻辑无关的适配逻辑时，可以使用适配器模式将适配的细节封装在适配器中，使客户端代码更简洁。

五、实例说明

类适配器模式

以充电器为例，充电器本身相当于适配器（Adapter），220V 交流电相当于被适配者（Adapted），而 5V 直流电是目标接口（ITarget）。适配器类通过继承 220V 类（或持有 220V 类的实例），实现 5V 类接口，完成从 220V 交流电到 5V 直流电的适配。这样，手机等需要 5V 直流电的设备就可以通过充电器（适配器）来充电了。

综上所述，适配器模式是一种强大的设计模式，它允许接口不兼容的类一起工作，并通过转换接口来满足客户端的需求。在实际开发中，可以根据具体场景选择合适的适配器类型来实现接口转换和类复用。
