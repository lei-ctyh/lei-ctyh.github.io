Obsidian Minimal 两个设置插件说明书

本文说明 Obsidian 的 Minimal 主题中两个常用插件的职责、配置位置、推荐用法和避坑原则：

  • Minimal Theme Settings
  • Style Settings

适用场景：你正在使用 Minimal 主题，希望把 Obsidian 调成稳定、清爽、接近飞书文档的阅读体验。

1. 总体结论

Minimal 主题的配置不要全部写进 CSS，也不要只靠一个插件。推荐按三层管理：

层级 用途 典型配置 是否推荐
Obsidian 外观设置 Obsidian 原生基础配置 当前主题、基础字号、界面字体、正文字体、代码字体 推荐
Minimal Theme Settings Minimal 的基础能力和全局开关 配色方案、背景对比度、字号、行高、正文宽度、图片宽度 强烈推荐
Style Settings Minimal 暴露出的细节变量 标题颜色、链接色、引用块、代码块、表格、Callout 强烈推荐
CSS snippets 插件无法覆盖的最后补丁 Live Preview 引用块、图片圆角、局部插件冲突 谨慎使用

一句话原则：

能用 Minimal Theme Settings 调的，不写进 CSS；能用 Style Settings 调的，也不写进 CSS；CSS 只做最后 10% 的补丁。

2. Minimal Theme Settings 是什么

Minimal Theme Settings 是 Minimal 主题的官方配套插件。

它不是必须安装，但官方强烈推荐安装。它的作用是把 Minimal 的常用配置放到 Obsidian 设置面板里，让你不用写 CSS 就能调整主题的核心体验。

官方定位：

用来控制 Minimal 的颜色、字体和功能开关。

2.1 它适合管理什么

Minimal Theme Settings 更适合管理“大开关”和“全局体验”：

  • 配色方案
  • 背景对比度
  • 正文字号
  • 侧栏字号
  • 行高
  • 正文宽度
  • 宽屏正文宽度
  • 最大内容宽度
  • 图片/视频是否铺满正文宽度
  • 是否启用彩色标题
  • 是否启用彩色边框
  • 是否启用彩色选中状态
  • 是否显示导航标签
  • 是否开启 Focus mode
  • 是否显示链接下划线
  • 是否显示侧栏边框

2.4 配色方案

Minimal Theme Settings 内置一些常见配色方案：

  • Atom
  • Ayu
  • Catppuccin
  • Dracula
  • E-ink
  • Everforest
  • Flexoki
  • Gruvbox
  • macOS
  • Nord
  • Rosé Pine
  • Sky
  • Solarized
  • Things

如果你追求飞书文档那种低干扰、白底、浅灰、蓝色点缀的感觉，建议优先试：

  • Minimal 默认浅色
  • macOS
  • Flexoki
  • E-ink

不要一开始就选高饱和配色，否则会偏离飞书那种“轻、白、克制”的感觉。

2.5 背景对比度

Minimal Theme Settings 支持不同背景对比度。

浅色模式常见选项：

选项 效果 适合场景
Default 侧栏比主区域略深 Minimal 默认风格
Low contrast 侧栏和正文背景更接近 更接近飞书、Notion 的轻量界面
All white 侧栏和正文都更白 喜欢纯白文档感
High contrast 深色侧栏、浅色正文 更像传统 IDE 或强分区界面

如果你觉得左侧栏“灰灰暗暗”，优先尝试：

  1. Low contrast
  2. All white

不要优先用 CSS 强行改侧栏背景。

Minimal 官方说明里有一个关键点：

要让 Minimal 的 line width 设置生效，需要打开 Obsidian 编辑器设置里的 Readable line length。

3. Style Settings 是什么

Style Settings 是一个通用插件，不只服务 Minimal。

它会读取主题或 CSS snippet 里的 @settings 配置声明，然后生成可视化设置面板。

Minimal 对 Style Settings 支持非常完整，所以它可以调整很多 Minimal 的细节变量。

官方定位：

Minimal Theme Settings 负责基础自定义和快捷键；Style Settings 提供更深入的界面细节定制。

3.1 它适合管理什么

Style Settings 更适合管理“细节样式”：

  • 背景色
  • 边框色
  • 链接色
  • 标题字体、字号、字重、颜色
  • 引用块背景、文字、边框
  • 代码块背景、文字、语法高亮
  • 表格样式
  • Callout 样式
  • 图标颜色
  • 图片透明度
  • Dataview 显示样式
  • Graph 节点颜色
  • 标签颜色

4. 两个插件怎么分工

4.1 推荐分工表

需求 推荐位置 原因
主题选择 Obsidian 外观设置 原生配置最稳定
当前配色方案 Minimal Theme Settings Minimal 官方支持
背景对比度 Minimal Theme Settings 官方已有 Default/Low contrast/All white 等选项
正文字号 Minimal Theme Settings 全局排版基础项
正文行高 Minimal Theme Settings 官方配置，避免 CSS 冲突
正文宽度 Minimal Theme Settings 官方 line width 配置
图片是否铺满 Minimal Theme Settings 官方媒体宽度配置
标题颜色 Style Settings 每级标题可独立调
标题字号 Style Settings 每级标题可独立调
链接颜色 Style Settings Minimal 暴露了变量
引用背景 Style Settings Minimal 暴露了变量
引用圆角 CSS snippet Style Settings 通常不暴露圆角
代码块背景 Style Settings Minimal 暴露了变量
代码块边框/圆角 CSS snippet 细节补丁
左侧文件树当前项样式 CSS snippet 或 Minimal 功能 具体效果通常要 CSS 微调
插件样式冲突 CSS snippet 只能局部覆盖

4.2 不推荐的做法

不要这样做：

  • 直接改 themes/Minimal/theme.css
  • 同一个变量既在 Style Settings 里调，又在 CSS snippet 里调
  • 把所有视觉配置都塞进一个巨大 customize.css
  • 同时启用多个全局美化 snippet
  • 看到不满意就加 !important

这些做法短期有效，长期会导致你不知道到底是谁在生效。

5.3 CSS snippet 只保留这些

如果你追求飞书效果，CSS snippet 只建议保留：

  • 引用块圆角和内边距
  • Live Preview 中连续引用块的背景修正
  • 代码块圆角、边框、内边距
  • 图片圆角
  • 左侧文件树当前项蓝色高亮

不要把正文字号、正文宽度、行高、标题颜色写进 CSS snippet。

6.1 为什么我改了 line width 没效果

6.2 为什么左侧栏看起来灰灰暗暗

优先检查 Minimal Theme Settings 的 Background contrast。

建议顺序：

  1. 试 Low contrast
  2. 再试 All white
  3. 最后才考虑 CSS snippet

不要一开始就写 CSS 强行改侧栏背景。

6.4 为什么编辑模式和阅读模式不一致

Obsidian 的阅读模式和 Live Preview 使用的 DOM 结构不同。

Style Settings 通常能影响变量，但一些细节，例如连续引用块的圆角、> 符号隐藏、代码块内边距，可能需要 CSS snippet 补丁。

6.5 为什么 Style Settings 里改了，文件又被自动清理

Style Settings 会根据主题声明的 @settings 校验配置项。

如果你手动写入了主题没有声明的 key，插件可能会在加载后清理掉。

所以手动改 data.json 时，最好确认 Minimal 的 theme.css 里确实存在对应 id。

7. 当前库推荐维护规则

你的库建议这样维护：

文件 负责内容
.obsidian/appearance.json 当前主题、基础字号、字体、启用哪些 CSS snippets
.obsidian/plugins/obsidian-minimal-settings/data.json Minimal 的全局排版和功能开关
.obsidian/plugins/obsidian-style-settings/data.json Minimal 的颜色、标题、代码、引用等细节变量
.obsidian/snippets/feishu-doc-style.css 飞书风格的最后补丁
.obsidian/snippets/customize.css 你已有的个人杂项补丁，后续建议继续拆小

如果以后你想调整外观，推荐顺序是：

  1. 先看 Obsidian 原生设置能不能调。
  2. 再看 Minimal Theme Settings 能不能调。
  3. 再看 Style Settings 能不能调。
  4. 最后才写 CSS snippet。
